# Design contract

## Three-axis law

- Vertical movement changes relationship.
- Horizontal movement changes selection.
- Depth changes attention.

Each axis maps to explicit application state. No axis is reused for an unrelated action.

## State-selected atmosphere

Cabinet, drawer, observation, comparison, confidence, disturbance, placement, privacy, processing, and reduced-motion states select visual variants. Blur, metal luminance, orange infrastructure signals, rail registration, and drawer depth never create or upgrade evidence state.

## Motion

Finite carousel travel is damped and stops at both ends. Drawer opening uses restrained depth without spring overshoot. Disturbance introduces a second drawer edge and split signal field without warning shake. Progress Mode withdraws irrelevant hardware rather than zooming the whole screen. Re-shelving follows an explicit destination track. Placement resolves to a visible sealed state before record generation becomes available.

Reduced motion replaces travel with immediate state replacement, static offsets, explicit headings, and position labels. Every gesture has a button or movement-menu equivalent.

## Responsive behavior

The cabinet is not uniformly scaled. At 320 px, secondary metadata and privacy detail collapse first while product identity, drawer pull, position, and 44 px actions remain. Wider widths add reading space without enlarging all hardware. Safe-area insets and horizontal overflow prevention are part of the base shell.
